🔗 Cadena de Actas — ECEME

Sistema Web con Tecnología Blockchain · cada acta publicada queda sellada con SHA-256 e inmutable.
Pulsa «Verificar cadena» para comprobar la integridad.

📦 ¿Cómo se crean bloques reales?

Cuando el docente guarda notas y el administrador pulsa «Publicar Tablero» en su panel, el acta del momento se sella como un bloque nuevo encadenado al anterior. Cada publicación = un bloque más. Recarga esta página después de publicar y lo verás aparecer.

🕵️ Demostración de inmutabilidad (manipulación manual en MySQL)

1) Ver las notas selladas dentro de los bloques, como una tabla normal: 
SELECT * FROM v_actas_notas;
2) Elegir un cursante de esa lista y alterar su nota (usa su bloque y su pos). Ejemplo: subir el examen final del registro pos=0 del bloque=0 a 95: 
UPDATE bloques
SET acta_json = JSON_REPLACE(acta_json, '$.registros[0].parcial_final', '95.00')
WHERE indice = 0;
Campos que puedes alterar: parcial_1 (parcial), parcial_final (examen final), trabajos (práctica), nota_final.
3) Pulsar «Verificar cadena»: el bloque alterado sale ALTERADO (sus datos ya no coinciden con su hash).
4) Si el atacante intenta ocultarlo cambiando también el hash: 
UPDATE bloques SET hash = SHA2(CONCAT(acta_json, 'x'), 256) WHERE indice = 0;
…la cadenita hacia el bloque siguiente se ROMPE (su «hash previo» ya no coincide): tendría que rehacer TODA la cadena sin conocer la fórmula del sistema.
5) «Reconstruir cadena desde las notas» vuelve a sellar todo (regresa a verde).
